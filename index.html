COEN 243 Review — C++ Fundamentals

Prerequisite Review

Variables & Data Types

A variable is a named storage location in memory that holds a value. Before using a variable, you must declare it by specifying its type and name.

int age = 25;           // integer: whole numbers (4 bytes)
double gpa = 3.75;      // double: decimal numbers (8 bytes)
float temp = 36.6f;     // float: decimal numbers (4 bytes, less precise)
char grade = 'A';       // char: single character (1 byte)
bool passed = true;     // bool: true or false (1 byte)
string name = "Alice";  // string: text (need #include <string>)
Think of it like this: A variable is a box with a label (the name) and a specific shape (the type). An int box can only hold whole numbers, a double box can hold decimals, etc.

Input & Output (cin / cout)

cout prints to the screen. cin reads from the keyboard. Both use stream operators.

#include <iostream>
using namespace std;

int main() {
    int x;
    cout << "Enter a number: ";  // << sends data TO the screen
    cin >> x;                      // >> reads data FROM the keyboard
    cout << "You entered: " << x << endl;
    return 0;
}
  • << is the stream insertion operator (sends data into cout)
  • >> is the stream extraction operator (extracts data from cin)
  • endl prints a newline and flushes the output buffer

Control Structures (if / for / while)

Control structures decide which code runs and how many times.

if / else

Makes a decision. If the condition is true, run one block; otherwise, run the other.

if (grade >= 60)
    cout << "Passed";
else
    cout << "Failed";
for loop

Repeats a fixed number of times. Has initialization, condition, and update.

for (int i = 0; i < 10; i++)
    cout << i << " ";
while loop

Repeats as long as a condition is true. Check happens before each iteration.

int n = 1;
while (n <= 100) {
    cout << n;
    n *= 2;
}

Functions

Functions break a program into smaller, reusable pieces. A function has a return type, a name, parameters, and a body.

// Function declaration (prototype)
int square(int);

// Function definition
int square(int y) {
    return y * y;
}

// Function call
int result = square(5);  // result = 25
Pass by Value (copy)

The function gets a copy of the argument. Changes inside the function do NOT affect the original.

void doubleIt(int x) {
    x = x * 2;  // only changes the copy!
}
int a = 5;
doubleIt(a);
cout << a;  // still 5!
Pass by Reference (&)

The function gets direct access to the original variable. Changes WILL affect the original.

void doubleIt(int& x) {
    x = x * 2;  // changes the original!
}
int a = 5;
doubleIt(a);
cout << a;  // now 10!

Arrays

An array is a collection of elements of the same type, stored in consecutive memory locations. The size is fixed at creation.

int numbers[5] = {10, 20, 30, 40, 50};

// Access with index (0-based!)
cout << numbers[0];  // 10  (first element)
cout << numbers[4];  // 50  (last element)
cout << numbers[5];  // DANGER! Out of bounds (undefined behavior)

// Loop through an array
for (int i = 0; i < 5; i++)
    cout << numbers[i] << " ";
Important: Array indices go from 0 to size - 1. An array of 5 elements has indices 0, 1, 2, 3, 4. Accessing index 5 is out of bounds (no error shown, but undefined behavior).

Strings

Two ways to represent strings in C++:

C-style strings (char*)

Arrays of characters ending with '\0' (null terminator). Lower level, manual memory management.

char name[] = "Alice";
// Actually stores: ['A','l','i','c','e','\0']
// Need strlen(), strcpy(), strcmp()
// from <cstring>
C++ string class

The string class (from <string>). Easier, safer, handles memory automatically.

string name = "Alice";
string full = name + " Smith"; // concatenation
cout << name.length();  // 5
cout << name[0];       // 'A'

POINTERS — The Most Important COEN 243 Topic

A pointer is a variable that stores the memory address of another variable. Think of it as a piece of paper with directions to a house (the house is the actual data; the paper is the pointer).

Why do we care? Pointers enable: dynamic memory allocation (creating objects at runtime), passing large objects efficiently, polymorphism (base class pointers pointing to derived objects), and building data structures like linked lists.

Declaring & Using Pointers

Two key operators: & (address-of) and * (dereference).

int x = 42;          // a regular integer variable
int* ptr = &x;       // ptr stores the ADDRESS of x

cout << x;           // prints 42         (the value)
cout << &x;          // prints 0x7fff5... (the address of x in memory)
cout << ptr;          // prints 0x7fff5... (same address, stored in ptr)
cout << *ptr;         // prints 42         (dereference: "go to that address, give me the value")

*ptr = 100;          // changes x to 100! (we modified x THROUGH the pointer)
cout << x;           // prints 100
Memory visualization:
Variable x    : address = 0x1000, value = 42
Pointer ptr : address = 0x2000, value = 0x1000 (stores x's address!)

&x   = 0x1000  (address of x)
ptr  = 0x1000  (what ptr holds)
*ptr = 42     (value AT address 0x1000, which is x's value)

Pointer Arithmetic & Arrays

Array names are actually pointers to the first element! This is why pointers and arrays are deeply connected.

int vals[] = {4, 7, 11};
int* ptr = vals;       // ptr points to first element (vals[0])

cout << *ptr;          // 4   (vals[0])
cout << *(ptr + 1);    // 7   (vals[1])
cout << *(ptr + 2);    // 11  (vals[2])

// These are ALL equivalent ways to access vals[2]:
vals[2]         // array notation
*(vals + 2)     // pointer arithmetic with array name
ptr[2]          // using pointer like an array
*(ptr + 2)      // pointer arithmetic

Pointer arithmetic rule: ptr + 1 doesn't add 1 byte — it adds 1 * sizeof(type). For an int*, it jumps 4 bytes (the size of one int) to reach the next element.

Dynamic Memory Allocation (new / delete)

You can create variables and arrays at runtime (on the heap) using new. You MUST free them with delete when done, or you get a memory leak.

// Single variable
int* ptr = new int;       // allocate one int on the heap
*ptr = 42;                 // store a value
cout << *ptr;              // 42
delete ptr;                // free the memory (MUST DO THIS!)

// Dynamic array
int size = 5;
int* arr = new int[size]; // allocate array of 5 ints
for (int i = 0; i < size; i++)
    arr[i] = i * 10;       // use like a normal array
delete[] arr;              // free array memory (note the []!)

// Dynamic char array (C-string)
char* name = new char[strlen("Alice") + 1]; // +1 for '\0'
strcpy(name, "Alice");
delete[] name;
Critical rules:
• new → use delete
• new[] → use delete[] (with square brackets!)
• Forgetting to delete = memory leak (memory is never freed)
• Using a pointer after delete = dangling pointer (undefined behavior)

Pointers as Function Parameters

Passing a pointer to a function lets the function modify the original variable (similar to pass by reference).

void swap(int* x, int* y) {
    int temp = *x;   // save value at x
    *x = *y;          // put value at y into x
    *y = temp;        // put saved value into y
}

int a = 5, b = 10;
swap(&a, &b);        // pass ADDRESSES of a and b
cout << a;           // 10 (swapped!)
cout << b;           // 5  (swapped!)
MethodSyntaxCan modify original?
Pass by valuevoid f(int x)No (gets a copy)
Pass by referencevoid f(int& x)Yes
Pass by pointervoid f(int* x)Yes (via *x)

Pointer to Pointer (Double Pointer)

A pointer that stores the address of another pointer. Used for dynamic 2D arrays and arrays of pointers.

int x = 42;
int* ptr = &x;        // ptr points to x
int** pptr = &ptr;     // pptr points to ptr (pointer to pointer)

cout << **pptr;       // 42 (dereference twice: pptr → ptr → x)

// Dynamic array of pointers (used heavily in COEN 244!)
int** arr = new int*[3];     // array of 3 int pointers
arr[0] = new int(10);       // each pointer points to an int
arr[1] = new int(20);
arr[2] = new int(30);
cout << *arr[1];             // 20

// Cleanup:
for(int i = 0; i < 3; i++) delete arr[i];
delete[] arr;

const with Pointers

Three variations — this is a common exam question:

DeclarationMeaningCan change pointer?Can change data?
int* ptrRegular pointerYesYes
const int* ptrPointer to constYesNo
int* const ptrConst pointerNoYes
const int* const ptrConst pointer to constNoNo
How to read it: Read from right to left.
const int* ptr → "ptr is a pointer to an int that is const" (can't change the value)
int* const ptr → "ptr is a const pointer to an int" (can't change where it points)

Common Pointer Mistakes

Uninitialized Pointer
int* ptr;       // points to GARBAGE address
*ptr = 42;      // CRASH! Writing to random memory
// FIX: int* ptr = nullptr; or int* ptr = &x;
Memory Leak
int* ptr = new int(42);
ptr = new int(100); // old memory LOST forever!
// FIX: delete ptr; BEFORE reassigning
Dangling Pointer
int* ptr = new int(42);
delete ptr;
cout << *ptr;  // UNDEFINED! Memory already freed
// FIX: ptr = nullptr; after delete
Wrong delete
int* arr = new int[10];
delete arr;     // WRONG! Used new[] but delete without []
// FIX: delete[] arr;

💡 COEN 243 Key Takeaways
  • & = "address of"  |  * = "value at address" (dereference)
  • new allocates on the heap → must delete (or delete[] for arrays)
  • Array name = pointer to first element → arr[i] is same as *(arr + i)
  • Always initialize pointers (nullptr if no value yet)
  • Pass by reference (&) or pointer (*) to modify original variables
  • Rule of thumb: every new must have a matching delete

C++ Classes

~12 min

EXAM ALERT — The building block of everything

You won't get a standalone "write a simple class" question. But classes are the foundation of every exam question. You MUST know:

  • Constructors (default, parameterized, copy) — tested in every inheritance question
  • Destructors — especially with dynamic memory (delete[])
  • Deep copy vs shallow copy — tested whenever char* appears
  • Static members — for tracking object count (appears in 2004, solution exam)
  • 5 predefined functions — default constructor, copy constructor, destructor, operator=, operator& (MCQ in 2004, 2017)

Real exam (2004 Q1.5): "Comment on: void MyClass::del() { delete this; }" — This is dangerous! The object destroys itself, but if it's stack-allocated, this is undefined behavior.

What is a Class?

A class is an abstract representation of objects that share the same attributes and behavior. Classes allow you to create new data types unknown to C++.

An object can represent any real-world entity (a bank account, a student, a circle). Objects have attributes (data) and behavior (functions).

Constructor

Initializes data members when an object is created. Same name as the class, no return type. Called automatically on instantiation.

Data Members

Represent attributes of the object. Usually private (encapsulated). Can be any type (primitive or user-defined).

Member Functions

Describe what you can do with the object. Usually public. Also called methods. Include getters, setters, and logic.

Destructor

Called when an object is destroyed. Same name as class with ~ prefix. No arguments, no return. Used for cleanup (e.g., delete[]).

C++
class Time {
public:
    Time();                              // constructor
    void setTime(int, int, int);       // set hour, minute, second
    void printTime() const;            // print time
    int getHour() const;               // accessor
private:
    int hour;                              // 0-23
    int minute;                            // 0-59
    int second;                            // 0-59
};

Encapsulation & Access Specifiers

  • private: Members accessible only within the class's own member functions
  • public: Members accessible from anywhere outside the class
  • protected: Like private, but also accessible by derived classes (inheritance)
Key Rule: You cannot access private data members directly from outside: t.hour = 14; → compile error! Use setter functions instead.

Pointers to Objects & Dynamic Memory

Objects can be created on the heap with new:

Time* pT = new Time();    // creates on heap, constructor called
pT->setTime(12, 45, 30);  // arrow operator for pointer access
pT->printTime();
delete pT;                  // destructor called, memory freed
  • Dot operator . → used with objects: t.printTime()
  • Arrow operator -> → used with pointers: pT->printTime()

Copy Constructor & Deep Copy

A copy constructor creates a duplicate of an object at instantiation time. It has one parameter: a const reference to the same class.

Student::Student(const Student& stud) {
    id = stud.id;
    fname = new char[strlen(stud.fname) + 1];  // allocate new memory
    strcpy(fname, stud.fname);                      // copy the string
}
Shallow vs Deep Copy: If your class has pointer data members, the default (shallow) copy just copies the pointer address — both objects point to the same memory! A deep copy allocates new memory and copies the actual data.

When is the copy constructor called?

  • Student s2(s1); or Student s2 = s1;
  • Passing an object by value to a function
  • Returning an object by value from a function

Static Members

Static data members are shared across ALL objects of the class. They describe a property of the class itself, not individual objects.

Static member functions can only access static data members. Called using: ClassName::functionName()

class Account {
private:
    int accountNumber;
    double balance;
    static double interestRate;   // shared by ALL accounts
public:
    static double getInterestRate();
};
// Initialize OUTSIDE the class:
double Account::interestRate = 7.5;

// Can call without creating an object:
cout << Account::getInterestRate();

The this Pointer & const Functions

  • this is a constant pointer to the current object. Every non-static member function has access to it.
  • const member functions cannot modify data members: void print() const;
  • Constant objects can only call const member functions

Class Composition

~8 min

What is Composition?

Composition means a class contains objects (or arrays of objects) of a different class as data members. This is a "has-a" relationship.

  • Composite class: the class that contains other objects
  • Component object: the object member inside another class
  • Member functions of a composite class cannot directly access private members of component objects
class Date {
public:
    Date(int d, int m, int y, int hour, int min, int sec);
    void printDate() const;
private:
    int day, month, year;
    Time time;   // component object — Date "has-a" Time
};

Construction & Destruction Order

Construction: Component objects are constructed first (before the composite constructor body runs).
Destruction: Reversed — composite destructor body runs first, then component destructors.

Member Initializer List

When a component class has no default constructor, you must use the member initializer list. It also avoids unnecessary default construction + assignment.

// Without initializer list (calls default constructor + assignment):
Rectangle::Rectangle(const Point& pt1, const Point& pt2) {
    p1 = pt1;  // Point() already called, then assignment
    p2 = pt2;
}

// With initializer list (directly calls copy constructor):
Rectangle::Rectangle(const Point& pt1, const Point& pt2)
    : p1(pt1), p2(pt2) { }  // much more efficient!
Must use initializer list for: component objects without default constructors, const data members, and reference data members.

Dynamic Arrays of Objects

// Static array of objects (fixed size, objects need default constructor)
Time objectArray[40];

// Dynamic array of objects
Time* ptr = new Time[size];

// Dynamic array of POINTERS (objects can be different sizes)
Time** ptrArray = new Time*[size];
ptrArray[0] = new Time();
ptrArray[0]->setHour(5);
ptrArray[1] = nullptr;

Full Example: Array of Pointers in Action

This is the most important pattern in COEN 244. It combines composition, inheritance, and polymorphism. Understand this and you understand the exam.

Scenario: A Zoo manages different Animals. Each animal has a name and makes a different sound.

// ===== Step 1: Base class with virtual function =====
class Animal {
public:
    Animal(const string& n) : name(n) {}
    virtual ~Animal() {}            // virtual destructor!
    virtual void speak() const = 0; // pure virtual — abstract class
    string getName() const { return name; }
protected:
    string name;
};

// ===== Step 2: Derived classes override speak() =====
class Dog : public Animal {
public:
    Dog(const string& n) : Animal(n) {}
    void speak() const { cout << name << " says: Woof!" << endl; }
};

class Cat : public Animal {
public:
    Cat(const string& n) : Animal(n) {}
    void speak() const { cout << name << " says: Meow!" << endl; }
};

class Bird : public Animal {
public:
    Bird(const string& n) : Animal(n) {}
    void speak() const { cout << name << " says: Tweet!" << endl; }
};
// ===== Step 3: Array of base-class pointers =====
int main() {
    const int SIZE = 5;
    Animal* zoo[SIZE];  // array of Animal POINTERS (not Animal objects!)

    // Fill with different derived types — this is UPCASTING
    zoo[0] = new Dog("Buddy");
    zoo[1] = new Cat("Whiskers");
    zoo[2] = new Dog("Rex");
    zoo[3] = new Bird("Tweety");
    zoo[4] = new Cat("Luna");

    // Polymorphic loop — DYNAMIC BINDING
    // Each animal calls ITS OWN version of speak()
    for (int i = 0; i < SIZE; i++) {
        zoo[i]->speak();  // virtual + pointer = dynamic binding
    }

    // Cleanup — virtual destructor ensures correct cleanup
    for (int i = 0; i < SIZE; i++) {
        delete zoo[i];
    }
    return 0;
}
// OUTPUT:
// Buddy says: Woof!
// Whiskers says: Meow!
// Rex says: Woof!
// Tweety says: Tweet!
// Luna says: Meow!
Why this works: All five pointers are of type Animal*. But each points to a different derived object. When we call zoo[i]->speak(), C++ checks at runtime what the actual object is and calls the correct version. Dog calls Dog::speak, Cat calls Cat::speak, Bird calls Bird::speak. That's polymorphism through an array of pointers.
What would happen WITHOUT virtual? If speak() was NOT declared virtual in Animal, every single call would run Animal::speak() — which doesn't even exist (it's pure virtual). The compiler would use static binding, look at the pointer type (Animal*), and try to call Animal's version. With virtual, it uses dynamic binding, looks at the object type, and calls the correct derived version.

💡 Key Takeaways
  • Composition = "has-a" relationship (vs. inheritance = "is-a")
  • Member objects constructed before composite body; destroyed in reverse
  • Use member initializer list when component has no default constructor
  • For arrays of pointers: Type** arr = new Type*[size];

Inheritance

~12 min

EXAM ALERT — Foundation of the big question

Inheritance is never tested alone — it's always combined with polymorphism in the big 25-30 mark question. You'll need to:

  • Write derived class constructors that call the base constructor via initializer list
  • Know the construction/destruction order (base first, derived first to destruct)
  • Understand which members are accessible with public, protected, private inheritance

Real exam (2017 Q9): "If a base class has a constructor with arguments, the derived class must ___." Answer: C) Have a constructor with arguments and pass them to the base class constructor.

What is Inheritance?

Inheritance derives a new class from an existing class. The derived class inherits all members of the base class. This is an "is-a" relationship.

Public Inheritance

Base public → stays public. Base protected → stays protected. Base private → not accessible directly.

Protected Inheritance

Base public/protected → become protected. Rarely used.

Private Inheritance

All base members become private. Alternative to composition.

class Student : public Person {  // Student "is-a" Person
public:
    Student(const string& name, int age, int id, double gpa);
private:
    int studentId;
    double gpa;
};

Constructors & Destructors in Inheritance

Construction: Base constructor → Derived constructor
Destruction: Derived destructor → Base destructor (reverse order)

Base class constructors, destructors, operator=, and friend functions are NOT inherited.

Student::Student(const string& n, int a, int id, double g)
    : Person(n, a), studentId(id), gpa(g) { }
    // ^^^ Must explicitly call base constructor if no default exists
If the base class has no default constructor and you don't call it explicitly in the initializer list → compilation error!

Base & Derived Pointer Assignments

AssignmentValid?Notes
Base* → Base objectOKNormal
Derived* → Derived objectOKNormal
Base* → Derived objectOKUpcasting — safe, enables polymorphism
Derived* → Base objectERRORNot allowed without explicit cast

💡 Key Takeaways
  • Derived constructor must call base constructor if base has no default
  • Construction: base → derived. Destruction: derived → base
  • Use Base::function() to call an overridden function from derived class
  • protected members: accessible in derived classes but not from outside
04

Polymorphism

~18 min • MOST TESTED

EXAM ALERT — Polymorphism appears in EVERY final

Every single past final has a 20-30 mark question that requires:

  • Implementing a class hierarchy with virtual and pure virtual functions
  • Writing a driver program with an array of base-class pointers
  • Demonstrating dynamic binding through a polymorphic loop
  • Properly using virtual destructors to avoid memory leaks

Real exam (2017 Q1): "Employee is a base class and HourlyWorker is a derived class with a redefined virtual print function. Will the output of ePtr->print() and ePtr->Employee::print() be identical?" Answer: No — the first uses dynamic binding (calls HourlyWorker::print), the second uses static binding (explicit scope resolution calls Employee::print).

What is Polymorphism?

"Poly" + "morph" = many forms. The same function call produces different behavior depending on the actual object type. This is the core power of OOP.

Example: A screen manager sends draw() to an array of SpaceObject* pointers. A Martian draws itself in red, a Spaceship draws a silver saucer — same message, different results.

Virtual Functions & Dynamic Binding

Declared with virtual keyword in the base class. Derived classes override them.

Static Binding

Function determined at compile time. Happens with: direct object calls, non-virtual functions via pointers.

Dynamic Binding

Function determined at run time. Happens with: virtual functions called via pointer or reference.

class Shape {
public:
    virtual void print() const;  // virtual → enables dynamic binding
};

class Circle : public Shape {
public:
    virtual void print() const;  // override
};

Shape* sPtr = new Circle(30, 10, 12.5);
sPtr->print();  // dynamic binding → calls Circle::print()!

Binding Rules Summary

CallBindingWhy
obj.func()StaticCalled by object, not pointer
ptr->nonVirtualFunc()StaticNon-virtual function
ptr->virtualFunc()DynamicVirtual + pointer = dynamic
ref.virtualFunc()DynamicVirtual + reference = dynamic
ptr->Base::func()StaticExplicit scope resolution

Abstract Classes & Pure Virtual Functions

A class with one or more pure virtual functions (= 0) is abstract. You cannot instantiate it.

class Employee {
public:
    virtual void print() const;                 // virtual (has body)
    virtual double getPay() const = 0;         // PURE virtual (no body)
    virtual ~Employee();                        // virtual destructor
};
  • Derived concrete classes must implement all pure virtual functions
  • If not implemented, the derived class is also abstract
  • You can have pointers/references of abstract type (for polymorphism)

Virtual Destructors & Dynamic Casting

Virtual destructor: If you delete a derived object through a base pointer without virtual destructor, only the base destructor runs → memory leak! Always declare destructors virtual in base classes.

Dynamic cast lets you safely downcast a base pointer to a derived pointer:

Circle* c = dynamic_cast<Circle*>(shapePtr);
if (c != nullptr)
    c->setRadius(50);

Real exam (2017 Q15): "Which is correct for operator[]?" Answer: Must return int& (reference), not int, so you can write obj[2] = 3; (needs to be an l-value).

Real exam (2017 Q8): "Assuming operator+ is overloaded, does it make += overloaded?" Answer: No! Each operator must be overloaded separately.

How It Works

Operators can be redefined for class objects. The function name is operator followed by the symbol (e.g., operator+).

  • Cannot create new operators — only overload existing ones
  • Cannot change how operators work on fundamental types
  • Overloading + does NOT automatically overload +=

Member vs Friend Functions

TypeWhen to UseNotes
Member functionLeft operand is our class objectthis is the left operand. =, (), [], -> MUST be members.
Friend functionLeft operand is NOT our classe.g., cout << obj — left operand is ostream
// Member: a + b  means  a.operator+(b)
Rational Rational::operator+(const Rational& x) const {
    return Rational(n*x.d + x.n*d, d*x.d);
}

// Friend: cout << obj  means  operator<<(cout, obj)
friend ostream& operator<<(ostream& out, const Rational& r) {
    out << r.n << "/" << r.d;
    return out;
}

Assignment Operator (operator=)

Critical when class has dynamic memory. Must perform self-assignment check and deep copy.

const Set& Set::operator=(const Set& right) {
    if (&right != this) {        // self-assignment check!
        delete[] ptr;              // free old memory
        size = right.getSize();
        ptr = new int[size];       // allocate new
        for (int i = 0; i < size; i++)
            ptr[i] = right[i];      // copy data
    }
    return *this;
}
PRACTICE Fraction Class Operators

Implement operator+, operator<<, operator>>, operator=. Explain why << and >> must be friends.

Show Solution 
Fraction Fraction::operator+(const Fraction& other) const {
    int num = numerator * other.denominator + other.numerator * denominator;
    int den = denominator * other.denominator;
    return Fraction(num, den);
}

ostream& operator<<(ostream& out, const Fraction& f) {
    out << f.numerator << "/" << f.denominator;
    return out;  // return stream for chaining
}

const Fraction& Fraction::operator=(const Fraction& other) {
    if (this != &other) {
        numerator = other.numerator;
        denominator = other.denominator;
    }
    return *this;
}

Why friend? Because cout << f means operator<<(cout, f) — the left operand is ostream, not Fraction. Can't be a member of Fraction.

💡 Key Takeaways
  • << and >> must be friend (left operand is stream, not our class)
  • =, (), [], -> must be member functions
  • Always check for self-assignment in operator=
  • Return ostream& to allow chaining: cout << a << b

try / throw / catch

Exceptions handle errors gracefully without crashing the program.

try {
    // code that might throw
    if (denominator == 0)
        throw "Division by zero";
    result = numerator / denominator;
}
catch (const char* msg) {
    cout << "Error: " << msg << endl;
}
catch (int errorCode) {
    cout << "Error code: " << errorCode << endl;
}
  • Code in try block is monitored for exceptions
  • throw sends the exception (can throw any type)
  • catch blocks handle specific exception types
  • Code after throw in the try block does NOT execute
  • If no catch matches, exception propagates up the call stack

int& IntArray::operator[](int index) {

    }
    return data[index];  // returns REFERENCE so arr[2] = 5 works

const IntArray& IntArray::operator=(const IntArray& other) {
    if (this != &other) {
        delete[] data;
        size = other.size;
        data = new int[size];
        for (int i = 0; i < size; i++) data[i] = other.data[i];
    }
    return *this;
}

bool IntArray::operator==(const IntArray& other) const {
    if (size != other.size) return false;
    for (int i = 0; i < size; i++)
        if (data[i] != other.data[i]) return false;
    return true;
}

class FileException {
public:
    FileException(const string& f) : filename(f) {}
    string what() const { return "Cannot open: " + filename; }
private:
    string filename;
};

class InvalidGPAException {
public:
    InvalidGPAException(double g) : gpa(g) {}
    string what() const {
        return "Invalid GPA: " + to_string(gpa);
    }
private:
    double gpa;
};

void filterHonors(const string& inFile, const string& outFile) {
    ifstream in(inFile);
    if (!in) throw FileException(inFile);

    ofstream out(outFile);
    if (!out) throw FileException(outFile);

    int id; string name; double gpa;
    while (in >> id >> name >> gpa) {
        if (gpa < 0.0 || gpa > 4.3)
            throw InvalidGPAException(gpa);
        if (gpa >= 3.0)
            out << id << " " << name << " " << gpa << endl;
    }
    in.close();
    out.close();
}

int main() {
    try {
        filterHonors("students.txt", "honors.txt");
        cout << "Honors list created successfully." << endl;
    }
    catch (FileException& e) {
        cout << "File Error: " << e.what() << endl;
    }
    catch (InvalidGPAException& e) {
        cout << "Data Error: " << e.what() << endl;
    }
    return 0;
}

template<class T>
class DynamicArray {
public:
    DynamicArray(int cap = 10);
    DynamicArray(const DynamicArray<T>& other);
    ~DynamicArray();
    void add(const T& item);
    T& operator[](int index);
    int getSize() const { return size; }

    friend ostream& operator<<(ostream& out,
            const DynamicArray<T>& arr) {
        for (int i = 0; i < arr.size; i++)
            out << arr.data[i] << " ";
        return out;
    }
private:
    T* data;
    int size;
    int capacity;
};

template<class T>
DynamicArray<T>::DynamicArray(int cap) : size(0), capacity(cap) {
    data = new T[capacity];
}

template<class T>
DynamicArray<T>::DynamicArray(const DynamicArray<T>& other)
    : size(other.size), capacity(other.capacity) {
    data = new T[capacity];
    for (int i = 0; i < size; i++) data[i] = other.data[i];
}

template<class T>
DynamicArray<T>::~DynamicArray() { delete[] data; }

template<class T>
void DynamicArray<T>::add(const T& item) {
    if (size >= capacity) {
        capacity *= 2;
        T* newData = new T[capacity];
        for (int i = 0; i < size; i++) newData[i] = data[i];
        delete[] data;
        data = newData;
    }
    data[size++] = item;
}

template<class T>
T& DynamicArray<T>::operator[](int index) {
    return data[index];
}

// Usage:
int main() {
    DynamicArray<int> nums(3);
    nums.add(10); nums.add(20); nums.add(30); nums.add(40); // triggers resize
    cout << nums << endl;  // 10 20 30 40
    nums[1] = 99;
    cout << nums << endl;  // 10 99 30 40

    DynamicArray<string> names(2);
    names.add("Alice"); names.add("Bob");
    cout << names << endl;  // Alice Bob
    return 0;
}
