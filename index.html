Variables & Data Types

A variable is a named storage location in memory that holds a value. Before using a variable, you must declare it by specifying its type and name.

int age = 25;           // integer: whole numbers (4 bytes)
double gpa = 3.75;      // double: decimal numbers (8 bytes)
float temp = 36.6f;     // float: decimal numbers (4 bytes, less precise)
char grade = 'A';       // char: single character (1 byte)
bool passed = true;     // bool: true or false (1 byte)
string name = "Alice";  // string: text (need #include <string>)
Think of it like this: A variable is a box with a label (the name) and a specific shape (the type). An int box can only hold whole numbers, a double box can hold decimals, etc.

Input & Output (cin / cout)

cout prints to the screen. cin reads from the keyboard. Both use stream operators.

#include <iostream>
using namespace std;

int main() {
    int x;
    cout << "Enter a number: ";  // << sends data TO the screen
    cin >> x;                      // >> reads data FROM the keyboard
    cout << "You entered: " << x << endl;
    return 0;
}
  • << is the stream insertion operator (sends data into cout)
  • >> is the stream extraction operator (extracts data from cin)
  • endl prints a newline and flushes the output buffer

Control Structures (if / for / while)

Control structures decide which code runs and how many times.

if / else

Makes a decision. If the condition is true, run one block; otherwise, run the other.

if (grade >= 60)
    cout << "Passed";
else
    cout << "Failed";
for loop

Repeats a fixed number of times. Has initialization, condition, and update.

for (int i = 0; i < 10; i++)
    cout << i << " ";
while loop

Repeats as long as a condition is true. Check happens before each iteration.

int n = 1;
while (n <= 100) {
    cout << n;
    n *= 2;
}

Functions

Functions break a program into smaller, reusable pieces. A function has a return type, a name, parameters, and a body.

// Function declaration (prototype)
int square(int);

// Function definition
int square(int y) {
    return y * y;
}

// Function call
int result = square(5);  // result = 25
Pass by Value (copy)

The function gets a copy of the argument. Changes inside the function do NOT affect the original.

void doubleIt(int x) {
    x = x * 2;  // only changes the copy!
}
int a = 5;
doubleIt(a);
cout << a;  // still 5!
Pass by Reference (&)

The function gets direct access to the original variable. Changes WILL affect the original.

void doubleIt(int& x) {
    x = x * 2;  // changes the original!
}
int a = 5;
doubleIt(a);
cout << a;  // now 10!

Arrays

An array is a collection of elements of the same type, stored in consecutive memory locations. The size is fixed at creation.

int numbers[5] = {10, 20, 30, 40, 50};

// Access with index (0-based!)
cout << numbers[0];  // 10  (first element)
cout << numbers[4];  // 50  (last element)
cout << numbers[5];  // DANGER! Out of bounds (undefined behavior)

// Loop through an array
for (int i = 0; i < 5; i++)
    cout << numbers[i] << " ";
Important: Array indices go from 0 to size - 1. An array of 5 elements has indices 0, 1, 2, 3, 4. Accessing index 5 is out of bounds (no error shown, but undefined behavior).

Strings

Two ways to represent strings in C++:

C-style strings (char*)

Arrays of characters ending with '\0' (null terminator). Lower level, manual memory management.

char name[] = "Alice";
// Actually stores: ['A','l','i','c','e','\0']
// Need strlen(), strcpy(), strcmp()
// from <cstring>
C++ string class

The string class (from <string>). Easier, safer, handles memory automatically.

string name = "Alice";
string full = name + " Smith"; // concatenation
cout << name.length();  // 5
cout << name[0];       // 'A'

POINTERS — The Most Important COEN 243 Topic

A pointer is a variable that stores the memory address of another variable. Think of it as a piece of paper with directions to a house (the house is the actual data; the paper is the pointer).

Why Pointers Matter for COEN 244 — The Big Picture

Pointers aren't just a 243 topic — they are the backbone of EVERY concept in 244. Here's how:

1. Dynamic Memory (Classes)

When a class has char* or int*, you allocate with new and free with delete. This triggers the Rule of Three — constructor, copy constructor, destructor. Every exam has this.

2. Object Pointers (Inheritance)

Student* s = new Student(...); — you create objects on the heap and access them with -> arrow operator. This is how you build dynamic collections of objects.

3. Array of Base Pointers (Polymorphism)

Shape* arr[10]; — an array where each pointer can point to a DIFFERENT derived type (Circle, Rectangle, Triangle). This is THE exam pattern. Without pointers, polymorphism doesn't work.

4. Double Pointers (Composition)

Item** items = new Item*[size]; — a dynamic array of pointers. Used when a manager class needs to store a flexible collection of objects that can be different types.

If you don't understand pointers, you can't do ANY of this:
• Can't create objects dynamically (new / delete)
• Can't implement deep copy (constructor, copy constructor, operator=)
• Can't do polymorphism (base class pointers pointing to derived objects)
• Can't manage collections (arrays of pointers with add/remove/search)
• Can't pass objects efficiently (pass by reference/pointer instead of value)
Pointers are not "one topic" — they are woven into EVERY topic.

The Pointer Journey: From Simple to Polymorphism

Here's the progression from a simple pointer all the way to the exam pattern, step by step:

Level 1 — Pointer to a single variable

int x = 42;
int* ptr = &x;      // ptr holds the address of x
cout << *ptr;        // 42 — dereference to get the value

Level 2 — Pointer to a single object

Student* s = new Student("Alice", 3.8);  // object on the heap
s->print();                               // arrow operator to access members
delete s;                                  // YOU must free it

Level 3 — Array of objects (all same type)

Student* arr = new Student[3];  // 3 Students on heap (needs default constructor)
arr[0].setName("Alice");          // dot operator (arr[0] IS an object)
delete[] arr;                     // delete array

Level 4 — Array of POINTERS (each can be different!)

Student** arr = new Student*[3]; // 3 POINTERS (not objects yet!)
arr[0] = new Student("Alice", 3.8); // now create each object
arr[1] = new Student("Bob", 3.5);
arr[2] = nullptr;                 // can be empty! (objects can't be nullptr)
arr[0]->print();                   // arrow operator (arr[0] is a POINTER)
for(int i=0; i<3; i++) if(arr[i]) delete arr[i]; // delete each object
delete[] arr;                     // then delete the array itself

Level 5 — Array of BASE CLASS pointers (POLYMORPHISM!)

Shape* shapes[3];                       // array of Shape POINTERS
shapes[0] = new Circle(5.0);          // Circle IS-A Shape → upcasting OK
shapes[1] = new Rectangle(4.0, 6.0); // Rectangle IS-A Shape → OK
shapes[2] = new Circle(3.0);          // another Circle → OK

for(int i = 0; i < 3; i++) {
    shapes[i]->area();   // DYNAMIC BINDING — calls Circle or Rectangle version!
    delete shapes[i];   // virtual destructor → correct cleanup
}
See the progression?
int* → Student* → Student*[] → Student** → Shape*[] (polymorphism)

Each level builds on the last. Level 5 is the exam pattern. But you can't understand Level 5 without understanding Levels 1-4 first. That's why pointers are the foundation of everything.

Declaring & Using Pointers

Two key operators: & (address-of) and * (dereference).

int x = 42;          // a regular integer variable
int* ptr = &x;       // ptr stores the ADDRESS of x

cout << x;           // prints 42         (the value)
cout << &x;          // prints 0x7fff5... (the address of x in memory)
cout << ptr;          // prints 0x7fff5... (same address, stored in ptr)
cout << *ptr;         // prints 42         (dereference: "go to that address, give me the value")

*ptr = 100;          // changes x to 100! (we modified x THROUGH the pointer)
cout << x;           // prints 100
Memory visualization:
Variable x    : address = 0x1000, value = 42
Pointer ptr : address = 0x2000, value = 0x1000 (stores x's address!)

&x   = 0x1000  (address of x)
ptr  = 0x1000  (what ptr holds)
*ptr = 42     (value AT address 0x1000, which is x's value)

Pointer Arithmetic & Arrays

Array names are actually pointers to the first element! This is why pointers and arrays are deeply connected.

int vals[] = {4, 7, 11};
int* ptr = vals;       // ptr points to first element (vals[0])

cout << *ptr;          // 4   (vals[0])
cout << *(ptr + 1);    // 7   (vals[1])
cout << *(ptr + 2);    // 11  (vals[2])

// These are ALL equivalent ways to access vals[2]:
vals[2]         // array notation
*(vals + 2)     // pointer arithmetic with array name
ptr[2]          // using pointer like an array
*(ptr + 2)      // pointer arithmetic

Pointer arithmetic rule: ptr + 1 doesn't add 1 byte — it adds 1 * sizeof(type). For an int*, it jumps 4 bytes (the size of one int) to reach the next element.

Dynamic Memory Allocation (new / delete)

You can create variables and arrays at runtime (on the heap) using new. You MUST free them with delete when done, or you get a memory leak.

// Single variable
int* ptr = new int;       // allocate one int on the heap
*ptr = 42;                 // store a value
cout << *ptr;              // 42
delete ptr;                // free the memory (MUST DO THIS!)

// Dynamic array
int size = 5;
int* arr = new int[size]; // allocate array of 5 ints
for (int i = 0; i < size; i++)
    arr[i] = i * 10;       // use like a normal array
delete[] arr;              // free array memory (note the []!)

// Dynamic char array (C-string)
char* name = new char[strlen("Alice") + 1]; // +1 for '\0'
strcpy(name, "Alice");
delete[] name;
Critical rules:
• new → use delete
• new[] → use delete[] (with square brackets!)
• Forgetting to delete = memory leak (memory is never freed)
• Using a pointer after delete = dangling pointer (undefined behavior)

Pointers as Function Parameters

Passing a pointer to a function lets the function modify the original variable (similar to pass by reference).

void swap(int* x, int* y) {
    int temp = *x;   // save value at x
    *x = *y;          // put value at y into x
    *y = temp;        // put saved value into y
}

int a = 5, b = 10;
swap(&a, &b);        // pass ADDRESSES of a and b
cout << a;           // 10 (swapped!)
cout << b;           // 5  (swapped!)
MethodSyntaxCan modify original?
Pass by valuevoid f(int x)No (gets a copy)
Pass by referencevoid f(int& x)Yes
Pass by pointervoid f(int* x)Yes (via *x)

Pointer to Pointer (Double Pointer)

A pointer that stores the address of another pointer. Used for dynamic 2D arrays and arrays of pointers.

int x = 42;
int* ptr = &x;        // ptr points to x
int** pptr = &ptr;     // pptr points to ptr (pointer to pointer)

cout << **pptr;       // 42 (dereference twice: pptr → ptr → x)

// Dynamic array of pointers (used heavily in COEN 244!)
int** arr = new int*[3];     // array of 3 int pointers
arr[0] = new int(10);       // each pointer points to an int
arr[1] = new int(20);
arr[2] = new int(30);
cout << *arr[1];             // 20

// Cleanup:
for(int i = 0; i < 3; i++) delete arr[i];
delete[] arr;

Common Pointer Mistakes

Uninitialized Pointer
int* ptr;       // points to GARBAGE address
*ptr = 42;      // CRASH! Writing to random memory
// FIX: int* ptr = nullptr; or int* ptr = &x;
Memory Leak
int* ptr = new int(42);
ptr = new int(100); // old memory LOST forever!
// FIX: delete ptr; BEFORE reassigning
Dangling Pointer
int* ptr = new int(42);
delete ptr;
cout << *ptr;  // UNDEFINED! Memory already freed
// FIX: ptr = nullptr; after delete

EXAM ALERT — The building block of everything

You won't get a standalone "write a simple class" question. But classes are the foundation of every exam question. You MUST know:

  • Constructors (default, parameterized, copy) — tested in every inheritance question
  • Destructors — especially with dynamic memory (delete[])
  • Deep copy vs shallow copy — tested whenever char* appears
  • Static members — for tracking object count (appears in 2004, solution exam)
  • 5 predefined functions — default constructor, copy constructor, destructor, operator=, operator& (MCQ in 2004, 2017)

Real exam (2004 Q1.5): "Comment on: void MyClass::del() { delete this; }" — This is dangerous! The object destroys itself, but if it's stack-allocated, this is undefined behavior.

What is a Class?

A class is an abstract representation of objects that share the same attributes and behavior. Classes allow you to create new data types unknown to C++.

An object can represent any real-world entity (a bank account, a student, a circle). Objects have attributes (data) and behavior (functions).

Constructor

Initializes data members when an object is created. Same name as the class, no return type. Called automatically on instantiation.

Data Members

Represent attributes of the object. Usually private (encapsulated). Can be any type (primitive or user-defined).

Member Functions

Describe what you can do with the object. Usually public. Also called methods. Include getters, setters, and logic.

Destructor

Called when an object is destroyed. Same name as class with ~ prefix. No arguments, no return. Used for cleanup (e.g., delete[]).

C++
class Time {
public:
    Time();                              // constructor
    void setTime(int, int, int);       // set hour, minute, second
    void printTime() const;            // print time
    int getHour() const;               // accessor
private:
    int hour;                              // 0-23
    int minute;                            // 0-59
    int second;                            // 0-59
};

Encapsulation & Access Specifiers

  • private: Members accessible only within the class's own member functions
  • public: Members accessible from anywhere outside the class
  • protected: Like private, but also accessible by derived classes (inheritance)
Key Rule: You cannot access private data members directly from outside: t.hour = 14; → compile error! Use setter functions instead.

Pointers to Objects & Dynamic Memory

Objects can be created on the heap with new:

Time* pT = new Time();    // creates on heap, constructor called
pT->setTime(12, 45, 30);  // arrow operator for pointer access
pT->printTime();
delete pT;                  // destructor called, memory freed
  • Dot operator . → used with objects: t.printTime()
  • Arrow operator -> → used with pointers: pT->printTime()

Copy Constructor & Deep Copy

A copy constructor creates a duplicate of an object at instantiation time. It has one parameter: a const reference to the same class.

Student::Student(const Student& stud) {
    id = stud.id;
    fname = new char[strlen(stud.fname) + 1];  // allocate new memory
    strcpy(fname, stud.fname);                      // copy the string
}
Shallow vs Deep Copy: If your class has pointer data members, the default (shallow) copy just copies the pointer address — both objects point to the same memory! A deep copy allocates new memory and copies the actual data.

When is the copy constructor called?

  • Student s2(s1); or Student s2 = s1;
  • Passing an object by value to a function
  • Returning an object by value from a function

Static Members

Static data members are shared across ALL objects of the class. They describe a property of the class itself, not individual objects.

Static member functions can only access static data members. Called using: ClassName::functionName()

class Account {
private:
    int accountNumber;
    double balance;
    static double interestRate;   // shared by ALL accounts
public:
    static double getInterestRate();
};
// Initialize OUTSIDE the class:
double Account::interestRate = 7.5;

// Can call without creating an object:
cout << Account::getInterestRate();

The this Pointer & const Functions

  • this is a constant pointer to the current object. Every non-static member function has access to it.
  • const member functions cannot modify data members: void print() const;
  • Constant objects can only call const member functions

Class Composition

~8 min

What is Composition?

Composition means a class contains objects (or arrays of objects) of a different class as data members. This is a "has-a" relationship.

  • Composite class: the class that contains other objects
  • Component object: the object member inside another class
  • Member functions of a composite class cannot directly access private members of component objects
class Date {
public:
    Date(int d, int m, int y, int hour, int min, int sec);
    void printDate() const;
private:
    int day, month, year;
    Time time;   // component object — Date "has-a" Time
};

Construction & Destruction Order

Construction: Component objects are constructed first (before the composite constructor body runs).
Destruction: Reversed — composite destructor body runs first, then component destructors.

Member Initializer List

When a component class has no default constructor, you must use the member initializer list. It also avoids unnecessary default construction + assignment.

// Without initializer list (calls default constructor + assignment):
Rectangle::Rectangle(const Point& pt1, const Point& pt2) {
    p1 = pt1;  // Point() already called, then assignment
    p2 = pt2;
}

// With initializer list (directly calls copy constructor):
Rectangle::Rectangle(const Point& pt1, const Point& pt2)
    : p1(pt1), p2(pt2) { }  // much more efficient!
Must use initializer list for: component objects without default constructors, const data members, and reference data members.

Dynamic Arrays of Objects

// Static array of objects (fixed size, objects need default constructor)
Time objectArray[40];

// Dynamic array of objects
Time* ptr = new Time[size];

// Dynamic array of POINTERS (objects can be different sizes)
Time** ptrArray = new Time*[size];
ptrArray[0] = new Time();
ptrArray[0]->setHour(5);
ptrArray[1] = nullptr;

Full Example: Array of Pointers in Action

This is the most important pattern in COEN 244. It combines composition, inheritance, and polymorphism. Understand this and you understand the exam.

Scenario: A Zoo manages different Animals. Each animal has a name and makes a different sound.

// ===== Step 1: Base class with virtual function =====
class Animal {
public:
    Animal(const string& n) : name(n) {}
    virtual ~Animal() {}            // virtual destructor!
    virtual void speak() const = 0; // pure virtual — abstract class
    string getName() const { return name; }
protected:
    string name;
};

// ===== Step 2: Derived classes override speak() =====
class Dog : public Animal {
public:
    Dog(const string& n) : Animal(n) {}
    void speak() const { cout << name << " says: Woof!" << endl; }
};

class Cat : public Animal {
public:
    Cat(const string& n) : Animal(n) {}
    void speak() const { cout << name << " says: Meow!" << endl; }
};

class Bird : public Animal {
public:
    Bird(const string& n) : Animal(n) {}
    void speak() const { cout << name << " says: Tweet!" << endl; }
};
// ===== Step 3: Array of base-class pointers =====
int main() {
    const int SIZE = 5;
    Animal* zoo[SIZE];  // array of Animal POINTERS (not Animal objects!)

    // Fill with different derived types — this is UPCASTING
    zoo[0] = new Dog("Buddy");
    zoo[1] = new Cat("Whiskers");
    zoo[2] = new Dog("Rex");
    zoo[3] = new Bird("Tweety");
    zoo[4] = new Cat("Luna");

    // Polymorphic loop — DYNAMIC BINDING
    // Each animal calls ITS OWN version of speak()
    for (int i = 0; i < SIZE; i++) {
        zoo[i]->speak();  // virtual + pointer = dynamic binding
    }

    // Cleanup — virtual destructor ensures correct cleanup
    for (int i = 0; i < SIZE; i++) {
        delete zoo[i];
    }
    return 0;
}
// OUTPUT:
// Buddy says: Woof!
// Whiskers says: Meow!
// Rex says: Woof!
// Tweety says: Tweet!
// Luna says: Meow!
Why this works: All five pointers are of type Animal*. But each points to a different derived object. When we call zoo[i]->speak(), C++ checks at runtime what the actual object is and calls the correct version. Dog calls Dog::speak, Cat calls Cat::speak, Bird calls Bird::speak. That's polymorphism through an array of pointers.
What would happen WITHOUT virtual? If speak() was NOT declared virtual in Animal, every single call would run Animal::speak() — which doesn't even exist (it's pure virtual). The compiler would use static binding, look at the pointer type (Animal*), and try to call Animal's version. With virtual, it uses dynamic binding, looks at the object type, and calls the correct derived version.

What is Inheritance?

EXAM ALERT — Polymorphism appears in EVERY final

Every single past final has a 20-30 mark question that requires:

  • Writing a driver program with an array of base-class pointers

Answer: No — the first uses dynamic binding (calls HourlyWorker::print), the second uses static binding (explicit scope resolution calls Employee::print).

What is Polymorphism?

"Poly" + "morph" = many forms. The same function call produces different behavior depending on the actual object type. This is the core power of OOP.

Example: A screen manager sends draw() to an array of SpaceObject* pointers. A Martian draws itself in red, a Spaceship draws a silver saucer — same message, different results.

Virtual Functions & Dynamic Binding

Declared with virtual keyword in the base class. Derived classes override them.

Static Binding

Function determined at compile time. Happens with: direct object calls, non-virtual functions via pointers.

Dynamic Binding

Function determined at run time. Happens with: virtual functions called via pointer or reference.

class Shape {
public:
    virtual void print() const;  // virtual → enables dynamic binding
};

class Circle : public Shape {
public:
    virtual void print() const;  // override
};

Shape* sPtr = new Circle(30, 10, 12.5);
sPtr->print();  // dynamic binding → calls Circle::print()!

Abstract Classes & Pure Virtual Functions

A class with one or more pure virtual functions (= 0) is abstract. You cannot instantiate it.

class Employee {
public:
    virtual void print() const;                 // virtual (has body)
    virtual double getPay() const = 0;         // PURE virtual (no body)

  • If not implemented, the derived class is also abstract
  • You can have pointers/references of abstract type (for polymorphism)

Virtual destructor: If you delete a derived object through a base pointer without virtual destructor, only the base destructor runs → memory leak! Always declare destructors virtual in base classes.

Operator Overloading

~12 min

EXAM ALERT — POST-MIDTERM • Always tested (15-20+ marks)

What they ask:

  • Implement operator<< (friend function) — appears in every final
  • Implement operator= with deep copy and self-assignment check — appears when class has char* or dynamic arrays
  • Implement comparison operators (<, ==) — 2011, 2017 finals
  • Implement operator[] — 2011 final, 2017 final

Real exam (2017 Q15): "Which is correct for operator[]?" Answer: Must return int& (reference), not int, so you can write obj[2] = 3; (needs to be an l-value).

Real exam (2017 Q8): "Assuming operator+ is overloaded, does it make += overloaded?" Answer: No! Each operator must be overloaded separately.

How It Works

In C++, operators like +, -, ==, << are actually function calls behind the scenes. When you write 5 + 3, the compiler knows how to add integers. But when you write student1 + student2, C++ has NO idea what + means for YOUR class. So you DEFINE it.

The function name is the keyword operator followed by the symbol: operator+, operator<<, operator=, operator[].

  • Cannot create new operators — only overload existing ones
  • Cannot change how operators work on fundamental types (int, double, etc.)
  • Overloading + does NOT automatically overload +=. Each operator is separate.

Member vs Friend — The #1 Rule

For every operator, ask yourself: "What is on the LEFT side?"

If the LEFT operand is YOUR class → make it a member function.
If the LEFT operand is NOT your class (like cout) → make it a friend function.

class Student {
private:
    int id;
    string name;
    double gpa;
public:
    Student(int i, string n, double g) : id(i), name(n), gpa(g) {}

    // ===== MEMBER: operator> (s1 > s2 → s1.operator>(s2)) =====
    // Left side is s1 — our class — so MEMBER
    bool operator>(const Student& other) const {
        return gpa > other.gpa;
    }

    // ===== MEMBER: operator== (s1 == s2 → s1.operator==(s2)) =====
    bool operator==(const Student& other) const {
        return id == other.id;
    }

    // ===== MEMBER: operator= (s1 = s2 → s1.operator=(s2)) =====
    const Student& operator=(const Student& other) {
        if (this != &other) {
            id = other.id; name = other.name; gpa = other.gpa;
        }
        return *this;
    }

    // ===== FRIEND: operator<< (cout << s1 → operator<<(cout, s1)) =====
    // Left side is cout (ostream) — NOT our class — so FRIEND
    friend ostream& operator<<(ostream& out, const Student& s) {
        out << "ID: " << s.id << " | " << s.name << " | GPA: " << s.gpa;
        return out;  // return stream for chaining: cout << a << b
    }

    // ===== FRIEND: operator>> (cin >> s1 → operator>>(cin, s1)) =====
    // Left side is cin (istream) — NOT our class — so FRIEND
    // NOTE: NO const on Student — we are WRITING into it
    friend istream& operator>>(istream& in, Student& s) {
        cout << "Enter ID, Name, GPA: ";
        in >> s.id >> s.name >> s.gpa;
        return in;
    }
};

int main() {
    Student s1(40123, "Alice", 3.8);
    Student s2(40456, "Bob", 3.5);

    if (s1 > s2)  cout << "Alice has higher GPA" << endl;
    cout << s1 << endl;   // ID: 40123 | Alice | GPA: 3.8
    cout << s2 << endl;   // ID: 40456 | Bob | GPA: 3.5

    Student s3(0, "", 0);
    cin >> s3;            // reads from keyboard into s3
    cout << s3 << endl;
}
Key difference between << and >>:
operator<< takes const Student& — we only READ from the object to print it.
operator>> takes Student& (NO const) — we WRITE into the object from keyboard input.

Assignment Operator (operator=) — Step by Step

When your class has dynamic memory (pointers), the default operator= does a shallow copy — both objects end up pointing to the same memory. You MUST write your own with a deep copy.

Why self-assignment check? Imagine a = a. Without the check:
Step 1: delete[] data — you just deleted YOUR OWN data!
Step 2: new char[strlen(other.data)+1] — but other IS you. other.data was just deleted. You're reading garbage.
The fix: if (this != &other) at the top. Same object? Skip everything. 
// The 5-step pattern — memorize this:
const MyClass& MyClass::operator=(const MyClass& other) {
    if (this != &other) {              // Step 1: self-assignment check
        delete[] data;                  // Step 2: free old memory
        size = other.size;               // Step 3: copy simple members
        data = new int[size];           // Step 4: allocate new memory
        for (int i = 0; i < size; i++)
            data[i] = other.data[i];     // Step 5: copy data element by element
    }
    return *this;                      // return *this for chaining: a = b = c
}

Exception Handling

EXAM ALERT — POST-MIDTERM • Expect more weight on the final

try / throw / catch — How It Works

When something goes wrong in your code (invalid input, file not found, division by zero), you have two choices: crash, or handle it gracefully. Exception handling lets you do the second.

Three keywords:

  • try — wrap code that MIGHT fail. You're saying "watch this code for problems."
  • throw — when something goes wrong, throw an exception. Like raising a flag: "PROBLEM HERE!"
  • catch — handle the problem. Each catch block specifies what TYPE of exception it handles.
try {
    // code that might throw
    if (denominator == 0)
        throw "Division by zero";   // THROW — everything after this is SKIPPED
    result = numerator / denominator; // this line NEVER runs if we threw
}
catch (const char* msg) {          // catches thrown strings
    cout << "Error: " << msg << endl;
}
catch (int errorCode) {              // catches thrown integers
    cout << "Error code: " << errorCode << endl;
}
// Program CONTINUES normally here after catch — it doesn't crash!
Critical flow rules:
1. The MOMENT throw happens, everything after it in the try block is SKIPPED.
2. Only the FIRST matching catch block runs. The rest are skipped.
3. If NO catch matches, the exception propagates up to the calling function. If nobody catches it, program terminates.
4. After the catch runs, the program continues normally. It doesn't crash.

File Streams — Reading and Writing Files

Up to now, all input comes from the keyboard (cin) and output goes to the screen (cout). File I/O lets you read from files and write to files instead. The beautiful thing: it works exactly like cin and cout — same >> and << operators.

Include <fstream>. Three classes:

ofstream (write)

output file stream. Data goes OUT from your program to the file. Use << just like cout.

ifstream (read)

input file stream. Data comes IN from the file to your program. Use >> just like cin.

fstream (both)

Can read AND write. Open with ios::in | ios::out.

Writing to a file:

  • Pass file streams by reference, not by value
  • get() reads one char; getline(file, str) reads a full line

Templates & STL

~8 min

EXAM ALERT — POST-MIDTERM • Expect more weight on the final

Templates/STL questions are typically shorter and easier. Common patterns:

  • 2004 Final Q2: Write a class template Point<T> with constructor, getSlope(), and operator<<
  • 2017 Final Q2: What does f1(2, 5) return given a function template? (Answer: 2 — template deduces T1=int)
  • 2017 Final Q10-11: Which destructor / isEmpty() syntax is correct for Stack<T>? (Must use template<class T> and Stack<T>::)
  • STL: "Determine the output" with vector push_back, sort, erase, iterators

What Are Templates? — The Problem They Solve

Imagine you need a function that finds the maximum of two values. Without templates, you'd write:

// WITHOUT templates — you write the SAME logic 3 times!
int maxInt(int a, int b) { return (a > b) ? a : b; }
double maxDouble(double a, double b) { return (a > b) ? a : b; }
string maxString(string a, string b) { return (a > b) ? a : b; }
// Same logic, different types. This is code duplication!

Templates fix this. You write the logic ONCE using a placeholder type called T. When you use it, the compiler replaces T with the actual type and generates the right version automatically.

// WITH templates — write it ONCE, works for ANY type
template<class T>
T findMax(T a, T b) { return (a > b) ? a : b; }

// The compiler generates versions for you:
findMax(10, 20);             // compiler sees int → generates int version → returns 20
findMax(3.14, 2.71);         // compiler sees double → generates double version → returns 3.14
findMax(string("Zebra"), string("Apple")); // string version → returns "Zebra"
Think of it like a cookie cutter. The template is the cutter shape. T is the type of dough. When you use int dough, you get an int cookie. When you use string dough, you get a string cookie. Same shape, different flavors. You made one cutter, not three.

Function Templates — Step by Step

A function template starts with template<class T> before the function. Then use T wherever you'd normally put a specific type.

Example 1: Print any array

template<class T>                              // T is a placeholder for ANY type
void printArray(const T* array, int count) {  // T* = pointer to array of T
    for (int i = 0; i < count; i++)
        cout << array[i] << " ";
    cout << endl;
}

int main() {
    int nums[] = {1, 2, 3, 4, 5};
    double gpas[] = {3.8, 3.5, 3.9};
    char letters[] = "HELLO";

    printArray(nums, 5);      // T = int    → prints: 1 2 3 4 5
    printArray(gpas, 3);      // T = double → prints: 3.8 3.5 3.9
    printArray(letters, 5);   // T = char   → prints: H E L L O
}

Example 2: Two template parameters

template<class T1, class T2>           // two different type placeholders
T1 f1(T1 value1, T2 value2) {
    if (value1 < value2) return value1;
    else return value1 * 2;
}

f1(2, 5);     // T1=int, T2=int. 2 < 5 → returns 2
f1(2, 1.1);   // T1=int, T2=double. 2 < 1.1? NO → returns 2*2 = 4
// NOTE: return type is T1 (int), so even though 1.1 is double, result is int!
// This exact question was on the 2017 final Q2
How does the compiler know what T is? It looks at the arguments you pass. If you call findMax(10, 20), it sees two ints, so T = int. If you call findMax(3.14, 2.71), it sees two doubles, so T = double. This is called type deduction — the compiler figures it out automatically.

Class Templates — What They Are

Just like function templates, you can make an entire class generic. Every data member and member function can use the placeholder type T.

Why? Imagine a Stack class. A Stack of integers, a Stack of doubles, a Stack of strings — the logic is identical. Only the data type changes. Instead of writing 3 separate Stack classes, you write ONE template.

Without Templates

You'd write StackInt, StackDouble, StackString — three classes with identical logic but different types. Terrible code duplication.

With Templates

You write Stack<T> ONCE. Then use Stack<int>, Stack<double>, Stack<string>. The compiler generates all three for you.

Class Template: Full Stack<T> Example

What is a Stack? A data structure that follows LIFO — Last In, First Out. Like a pile of plates: you put plates on top (push) and take plates from the top (pop). The last plate you put on is the first one you take off.

Step 1: The class declaration

template<class T>        // T = placeholder type (will become int, double, string, etc.)
class Stack {
private:
    int size;              // max capacity of the stack (always int)
    int top;               // index of the top element (-1 means empty)
    T* stackPtr;          // pointer to dynamic array of T
                           // If T=int → this is int*
                           // If T=string → this is string*
public:
    Stack(int s);          // constructor
    ~Stack();              // destructor
    void push(const T&);  // add element to top
    T pop();               // remove and return top element
    bool isEmpty() const; // check if stack is empty
    bool isFull() const;  // check if stack is full
};

Step 2: Implement member functions OUTSIDE the class

THE RULE: Every member function defined outside a class template needs TWO things:
1. template<class T> on the line above
2. Stack<T>:: not just Stack:: — you MUST include the <T>

WRONG: Stack::Stack(int s) { ... }
RIGHT: template<class T> Stack<T>::Stack(int s) { ... }
This was directly tested on the 2017 final (Q10-11). 
// Constructor
template<class T>               // ← line 1: template declaration
Stack<T>::Stack(int s) {      // ← line 2: Stack<T>:: (NOT Stack::)
    size = s;
    top = -1;                   // -1 means the stack is empty (no plates yet)
    stackPtr = new T[size];     // allocate array of T on the heap
}

// Destructor
template<class T>
Stack<T>::~Stack() {
    delete[] stackPtr;          // free the dynamic array (new[] → delete[])
}

// Push — add element to top of stack
template<class T>
void Stack<T>::push(const T& value) {
    if (!isFull()) {
        top++;                    // move top index up by 1
        stackPtr[top] = value;    // store the value at the new top
    } else
        cout << "Stack is full!" << endl;
}

// Pop — remove and return top element
template<class T>
T Stack<T>::pop() {
    if (!isEmpty()) {
        T value = stackPtr[top];  // grab the value at the top
        top--;                    // move top down by 1
        return value;             // return what we grabbed
    }
    cout << "Stack is empty!" << endl;
    return T();                  // return default value (0 for int, "" for string)
}

// isEmpty — check if stack has no elements
template<class T>
bool Stack<T>::isEmpty() const { return top == -1; }

// isFull — check if stack has reached capacity
template<class T>
bool Stack<T>::isFull() const { return top == size - 1; }

Step 0: How to Analyze an Exam Question

When you see a big question on the exam, follow this thinking process:

1. Identify the Hierarchy

What's the base class? What derives from it? Draw a tree on scrap paper. Look for "is-a" relationships.

2. Spot Dynamic Memory

Any char* or dynamic arrays? If yes → you need Rule of Three (constructor, copy constructor, destructor, operator=).

3. Find Pure Virtual

Which function makes NO sense in the base class? That's your = 0. It makes the base abstract.

4. Plan the Driver

Array of base pointers → fill with derived objects → polymorphic loop → delete. This is always the last part.

Let's apply this to a real scenario now. Read the problem below as if it's on your exam. Then follow along as we build the solution step by step.

The Problem: Airline Ticket System

An airline manages different types of tickets. All tickets share a passenger name, a flight number, and a base price. But the total cost is calculated differently for each type:

  • EconomyTicket — total cost = base price (no extras)
  • BusinessTicket — total cost = base price × 1.5 + lounge fee
  • FirstClassTicket — total cost = base price × 3.0 + suite fee + meal plan cost

The system must:

  • Track total tickets created (static counter)
  • Support operator<< to print ticket info
  • Support operator> to compare tickets by total cost
  • Throw an exception if base price is negative
  • Save ticket data to a file
  • Use an array of base-class pointers for polymorphic processing

Step 1: Identify the Hierarchy

Think: "What is-a what?" EconomyTicket IS-A Ticket. BusinessTicket IS-A Ticket. FirstClassTicket IS-A Ticket.

//         Ticket (abstract base)
//        /        |           \
// Economy    Business    FirstClass

Think: "Which function is different for each type?" → calculateCost(). An Economy calculates differently from Business which calculates differently from FirstClass. So calculateCost() should be pure virtual in the base.

Think: "Which function CAN have a default?" → print(). Every ticket has a name and flight number, so the base can print those. Derived classes can override to add more info.

Step 2: Define the Exception Class

Think: "What can go wrong?" → Negative price. Define a custom exception before the main classes.

class InvalidPriceException {
public:
    InvalidPriceException(double p) : price(p) {}
    string what() const {
        return "Invalid price: " + to_string(price) + ". Must be >= 0.";
    }
private:
    double price;
};

Step 3: Build the Base Class (Ticket)

Think: "What does EVERY ticket have?" → passenger name (char* for dynamic memory!), flight number, base price, static counter.

Think: "char* means I need Rule of Three" → constructor with deep copy, copy constructor, destructor, operator=.

Think: "virtual destructor because we'll delete through base pointers."

class Ticket {
public:
    // Constructor — throws if price negative
    Ticket(const char* name, int flight, double price) : flightNo(flight) {
        if (price < 0) throw InvalidPriceException(price);  // validate!
        basePrice = price;
        passengerName = new char[strlen(name) + 1];  // deep copy
        strcpy(passengerName, name);
        ticketCount++;
    }

    // Copy Constructor — deep copy
    Ticket(const Ticket& other)
        : flightNo(other.flightNo), basePrice(other.basePrice) {
        passengerName = new char[strlen(other.passengerName) + 1];
        strcpy(passengerName, other.passengerName);
        ticketCount++;
    }

    // Virtual Destructor — MUST be virtual for polymorphism
    virtual ~Ticket() {
        delete[] passengerName;
        ticketCount--;
    }

    // Pure Virtual — each ticket type calculates differently
    virtual double calculateCost() const = 0;

    // Virtual print — has a default, derived classes can override
    virtual void print() const {
        cout << passengerName << " | Flight " << flightNo
             << " | Base: $" << basePrice;
    }

    // operator> — compare by total cost (MEMBER: left side is our class)
    bool operator>(const Ticket& other) const {
        return calculateCost() > other.calculateCost();
    }

    // operator<< — FRIEND because left side is ostream
    friend ostream& operator<<(ostream& out, const Ticket& t) {
        t.print();  // calls the correct virtual print!
        out << " | Total: $" << t.calculateCost();
        return out;
    }

    // Getters
    const char* getName() const { return passengerName; }
    int getFlightNo() const { return flightNo; }
    double getBasePrice() const { return basePrice; }
    static int getTicketCount() { return ticketCount; }

protected:                          // protected so derived classes can access
    char* passengerName;
    int flightNo;
    double basePrice;
    static int ticketCount;
};

int Ticket::ticketCount = 0;         // static init OUTSIDE class
Checklist for the base class:
✓ Dynamic memory (char*) → deep copy in constructor + copy constructor
✓ Virtual destructor → for safe polymorphic deletion
✓ Pure virtual calculateCost() = 0 → makes class abstract
✓ Virtual print() with default body → derived can override
✓ Static counter → initialized outside class
✓ Protected members → derived classes can access
✓ operator> as member → left side is our class
✓ operator<< as friend → left side is ostream

Structure 1: Class with Dynamic Memory (Rule of Three)

Any time a class has char* or a dynamic array, you NEED all three: constructor (deep copy), copy constructor (deep copy), destructor (delete), plus operator=.

class Student {
public:
    // ---- Constructor (deep copy pattern) ----
    Student(const char* n, int i) : id(i) {
        name = new char[strlen(n) + 1];  // allocate (+1 for '\0')
        strcpy(name, n);                    // copy the string
    }

    // ---- Copy Constructor (SAME deep copy pattern) ----
    Student(const Student& other) : id(other.id) {
        name = new char[strlen(other.name) + 1];
        strcpy(name, other.name);
    }

    // ---- Destructor ----
    ~Student() {
        delete[] name;  // free dynamic memory (new[] → delete[])
    }

    // ---- operator= (assignment with deep copy) ----
    const Student& operator=(const Student& other) {
        if (this != &other) {           // 1. self-assignment check
            delete[] name;               // 2. free old memory
            id = other.id;                // 3. copy simple members
            name = new char[strlen(other.name) + 1]; // 4. allocate new
            strcpy(name, other.name);    // 5. copy data
        }
        return *this;                  // 6. return for chaining a = b = c
    }

private:
    char* name;  // dynamic memory → triggers Rule of Three
    int id;
};

Structure 4: Every Operator Overloading Pattern

class MyClass {
private:
    int* data;
    int size;

public:
    // ======== MEMBER OPERATORS (left side = our class) ========

    // operator+ (returns a NEW object)
    MyClass operator+(const MyClass& other) const {
        // create result, combine data, return result
    }

    // operator== (comparison, returns bool)
    bool operator==(const MyClass& other) const {
        if (size != other.size) return false;
        for (int i = 0; i < size; i++)
            if (data[i] != other.data[i]) return false;
        return true;
    }

    // operator> (comparison, returns bool)
    bool operator>(const MyClass& other) const {
        return size > other.size;
    }

    // operator= (DEEP COPY — when class has pointers)
    const MyClass& operator=(const MyClass& other) {
        if (this != &other) {           // 1. self-assignment check
            delete[] data;               // 2. free old
            size = other.size;            // 3. copy scalars
            data = new int[size];        // 4. allocate new
            for (int i = 0; i < size; i++)
                data[i] = other.data[i];  // 5. copy elements
        }
        return *this;                  // 6. return for chaining
    }

    // operator[] (subscript — MUST return reference)
    int& operator[](int index) {
        return data[index];  // int& so obj[2] = 5 works
    }

    // ======== FRIEND OPERATORS (left side ≠ our class) ========

    // operator<< (output — left side is ostream)
    friend ostream& operator<<(ostream& out, const MyClass& obj) {
        for (int i = 0; i < obj.size; i++)
            out << obj.data[i] << " ";
        return out;  // return for chaining cout << a << b
    }

    // operator>> (input — left side is istream)
    // NOTE: no const on MyClass — we are writing INTO it
    friend istream& operator>>(istream& in, MyClass& obj) {

class Manager {
public:
    // Constructor: allocate array of pointers, init to nullptr
    Manager(int max) : maxSize(max), count(0) {
        items = new Item*[maxSize];
        for (int i = 0; i < maxSize; i++)
            items[i] = nullptr;
    }

    // Destructor: delete each Item, then delete the array itself
    ~Manager() {
        for (int i = 0; i < count; i++)
            delete items[i];       // delete each object
        delete[] items;             // delete the array of pointers
    }

    // Add: check not full, check no duplicate, copy construct
    bool add(const Item& item) {
        if (count >= maxSize) return false;
        items[count] = new Item(item);  // uses copy constructor
        count++;
        return true;
    }

    // Remove: find, shift array left, decrement count
    bool remove(int id) {
        for (int i = 0; i < count; i++) {
            if (items[i]->getId() == id) {
                delete items[i];
                for (int j = i; j < count - 1; j++)
                    items[j] = items[j + 1];  // shift left
                count--;
                return true;
            }
        }
        return false;
    }

private:
    Item** items;    // dynamic array of Item pointers
    int maxSize;
    int count;
};

Write a class template DynamicArray<T> that works like a simplified vector:

  • add(const T&) adds an element (resize if full)

template<class T>
class DynamicArray {
public:
    DynamicArray(int cap = 10);
    DynamicArray(const DynamicArray<T>& other);
    ~DynamicArray();
    void add(const T& item);
    T& operator[](int index);
    int getSize() const { return size; }

    friend ostream& operator<<(ostream& out,
            const DynamicArray<T>& arr) {
        for (int i = 0; i < arr.size; i++)
            out << arr.data[i] << " ";
        return out;
    }
private:
    T* data;
    int size;
    int capacity;
};

template<class T>
DynamicArray<T>::DynamicArray(int cap) : size(0), capacity(cap) {
    data = new T[capacity];
}

template<class T>
DynamicArray<T>::DynamicArray(const DynamicArray<T>& other)
    : size(other.size), capacity(other.capacity) {
    data = new T[capacity];
    for (int i = 0; i < size; i++) data[i] = other.data[i];
}

template<class T>
DynamicArray<T>::~DynamicArray() { delete[] data; }

template<class T>
void DynamicArray<T>::add(const T& item) {
    if (size >= capacity) {
        capacity *= 2;
        T* newData = new T[capacity];
        for (int i = 0; i < size; i++) newData[i] = data[i];
        delete[] data;
        data = newData;
    }
    data[size++] = item;
}

template<class T>
T& DynamicArray<T>::operator[](int index) {
    return data[index];
}

int main() {
    DynamicArray<int> nums(3);
    nums.add(10); nums.add(20); nums.add(30); nums.add(40); // triggers resize
    cout << nums << endl;  // 10 20 30 40
    nums[1] = 99;
    cout << nums << endl;  // 10 99 30 40

    DynamicArray<string> names(2);
    names.add("Alice"); names.add("Bob");
    cout << names << endl;  // Alice Bob
    return 0;
}
